Prompt-based Learning in NLP

Posted on 2021-10-25 Edited on 2021-10-30 In Paper Reading

This blog is based on: Liu et al. (2021) Pre-train, Prompt, and Predict: A Systematic Survey of Prompting Methods in Natural Language Processing.

Introduction

In a traditional supervised learning system (e.g. text classification or machine translation), we take an input x, and predict an output y based ona model \(P(\textbf{y}|\textbf{x};\theta)\). The main issue with supervised learning tasks is the requirement of larger amount of annotated label, which are usually not suffcicient for many task. This can be addressed by prompt-based learning. Instead of learning \(P(\textbf{y}|\textbf{x};\theta)\), prompt-based learning attempts to learn the probablity \(P(\textbf{x};\theta)\) and use this probability to predict y.

After appropriate prompt engineering, label y is usually included in the prompt text.

Prompting Basics

Basic prompting works in following three steps.

Prompt Addition

In this step, a prompting function \(f_{\textit{prompt}}\) is applied to modify the input text x into a prompt \(x^{\prime}=f_{\textit{prompt}}(\textbf{x})\). Traditionally, the function consists of two steps: 1. Apply a template, which is a textual string that takes two slots: an input slot \([X]\) for input x and an answer slot \([Z]\) for an intermediate generated answer text z that will later be mapped into target y. 2. Fill slot \([X]\) with the input text x.

For example, in the case of sentiment analysis with x ='I love this movie', the template can be :"[X] Overall, it was a [Z] movie". Then, \(x^{\prime}\) will turn into "I love this movie. Overall it was a [Z] moview". In the context of machine translation. it may take the form as "Enligh: [X] French [Z]". > Cloze prompt: the answer slot is in the middle of the text. Prefix prompt: the input text comes entirely before z.

Answer Search

We search over the set of potential answers z by calculating the probability of their corresponding filled prompts using a pre-trainied LM \(P(\dot;\theta)\): \[\hat{\boldsymbol{z}}=\operatorname{search}_{\boldsymbol{z} \in \mathcal{Z}} P\left(f_{\mathrm{fill}}\left(\boldsymbol{x}^{\prime}, \boldsymbol{z}\right) ; \theta\right) \] where \(\mathcal{Z}\) is a set of all permissible values for slot [Z], \(f_{\mathrm{fill}}\) is a function that fills in the location [Z] in prompt \(x^{\prime}\) with the potential answer z. The search function could be an argmax or \(sampling\).

Answer Mapping

The next step is obtain the highest score output \(\hat{\textbf{y}}\) from the highest score \(\hat{\textbf{z}}\). In language generation tasks, the answer itself is the output. But in sentimen analysis tasks, multiple answers could result in the same output.

Pre-trained models

A wide variety of pre-trained LMs can be used to calculate \(P(\textbf{x};\theta)\). They can be summarized according to different perspectives.

Training Objectives

The main training objective of a pre-trained LM usually consists of some sort of objective predicting the probablity of text x. In a Standard Language Model (SLM), the objective is to optimize the probablity \(P(\textbf{x})\) of text from a training corpus. A popular alternative to standard LM objectives are denoising objectives. A noised sentence is first generated by applying some kind of noising function: \(\tilde{x}=f_{\mathrm{noise}}(\textbf{x})\), then the model is trained to predict the original input given this noised text \(P(\textbf{x}|\tilde{\textbf{x}})\).

Noising Functions

In literature, different noising functions have been applied to obatain the noised text \(\tilde{x}\) and the choice of noising functions has an effect on algorithm effcacy.

  • Masking (e.g. Devlin et al 2019) The text will be masked in different levels, replacing a token or multi-token span with a special token such as [MASK]. Masking can either be random from some distribution of specifically designed in introduce prior knowledge.
  • Replacement (e.g. Raffle et al. 2020) In stead of replaced with [MASK], the token is replace with another token or piece of information.
  • Deletion (e.g. Lewis et al. 2020a) Tokens will be deleted without adding [MASK] of any other toekn.
  • Permuation (eg. Liu et al. 2020a) The text is first divided into different spans (tokens, sub-sentential spans, or sentences), and then these spans are be permuted into a new text.

Directionality

The directionality of the calculation of representations can be generally divided into three ways.

  • Left-to-Right The representation of each word is calculated based on the word itsetl and all previous words in the sentence (Autogressive fasion).
  • Bidirectional The representation of each word is calculated based on all words in the sentence, including words to the left of the current word.
  • Hybrid Sometimes researches choose to mix the two above strategies in a single modle or condition the representations in a randomly permuted order. The conditioning is generally implemented through attention masking.

Typical Pre-training Methods

Based on different combinations of objective, noising function, and directionality, four pre-training methods can be introduced.

Left-to-Right Language Models a variety of auto-regressive LMs, which predicts the upcoming words or assign a probability \(P(\textbf{x})\) to a sequence of words \(\textbf{x}=x_1,x_2,\dots,x_n\). Accoding to the chain rule, we have: \[P(\textbf{x})=P(x_1)P(x_2|x_1) \dots P(x_n|x_1 \dots x_{n-1})\]. Representative L2R LMs including GPT-3 (Brown et al. 2020) and GPT-Neo (Black et al. 2021).

Masked Language Models (MLM) aims to predict masked text pieces based on surrounded context, e.g. the probability of word \(x_i\) given its context \(P(x_i|x_1,\dots,x_{i-1},x_{i+1},\dots,x_n)\).

Representative pre-trained models using MLMs include: BERT (Devlin et al., 2019), ERNIE (Zhang et al., 2019; Sun et al., 2019b) and many variants. In prompting methods, MLMs are generally most suitable for natural language understanding or analysis tasks (e.g., text classification, natural language inference , and extractive question answering). These tasks are often relatively easy to be reformulated into cloze problems, which are consistent with the training objectives of the MLM. Additionally, MLMs have been a pre-trained model of choice when exploring methods that combine prompting with fine-tuning,

Prefix LM The prefix LM is a left-to-right LM that decodes y conditioned on a prefixed sequence x, which is encoded by the same model parameters but with a fully-connected mask. Example: UniLM 1-2 (Dong et al. 2019; Bao et al. 2020)

Encoder-decoder uses a left-to-right LM to decode y conditioned on a separate encoder for text x with a fully-connected mask; the parameters of the encoder and decoder are not shared. Example: T5 (Raffel et al. 2020) and BART (Lewis et al. 2020a).

Prompt Engineering

Prompt engineering is the process of creating (by a human engineer or algorithm) a prompting function \(f_{\mathrm{prompt}}(\textbf{x})\) which produces the most effective performance on the downstream task. One must first consider the prompt shape, and then decide whether to take a manual or automated approach to create prompts of the desired shape.

Prompt shape There are two main varieties of prompts: cloze prompt, which fill in the blanks of a textual string and prefix prompt, which continue a string prefix. Prefix prompts is more appropriate for generation tasks or auto-regressive tasks. Cloze prompts are more appropriate for masked LMs.

Manual Template Engineering Create initutive templates manuanlly, which can be time consuming and requires lots of experience.

Automated Template Learning Many methods have been proposed to automate the template designe process. This worths a separate subsection.

Automated Template Learning

Automated template learning can be further categoriezed into discrete prompts, where the prompt is an actual text string, and contiuous prompts, where the prompt is instead described directly in the embedding space of the underlying LM.

Discrete prompts

Discrete prompts learning searchs for templates described in a discrete space, usually corresponding to natural language phrases.

  1. Mining, a mining-based method to automatically find templates given a set of training inputs x and outputs y. > How is the mining performed? See this paper.

  2. Prompt paraphrasing This approach takes in a given seed prompt (e.g. manually constructed or mined), and paraphrases it into a set of other candidate prompts. Typical paraphrasing methods include round-trip translation (translating the prompt into another language and then back), replacement from a thesaurus or using a neural prompt re-writer. > The neural writer looks interesting, check it here.

  3. Gradient-based Search Wallace et al. (2019) applied a gradient-based search over actual tokens to find short sequence sthat can trigger underlying pre-trained LM to generate the desired target prediction.

  4. Prompt Generation The generation of prompy can be considered as a task generation task and can be performed using standard natural language text generation model. For example, Gao et al. (2020) introduced the pre-traiend model T5 to generate template tokens based on pre-specified positions of where to insert template tokens.

Continuous prompts

In continuous prompts engineering, two constraines are removed: 1) the constraint of embeddings of template words be the embeddings of natural language (e.g. English) and 2) the constraint of the template being parameterized by the pre-trained LMs' parameters, i.e. templates can have their own parameters to be tuned for downstream tasks.

A question here, when we have continuos prompts, how can we map the answer?

  1. Tuning Initialized with Discrete Prompts The search for a continuous prompt is initialized from previously generated discrete prompts.

Answer Engineering

In contrast to prompt engineering, which designs appropriate inputs for prompting methods, answer engineering aims to search for an answer space \(\mathcal{Z}\) and a mapping function to the original output \(\mathcal{Y}\) that results in an effective predictive model. To perform answer engineering, one needs to first decide the answer shape and then an answer designmethod.

The choices of answer shape include:

  1. Tokens: One of the tokens in the pre-trained LM’s vocabulary, or a subset of the vocabulary. Usually used for text classification, relation extraction and named entity recognition.
  2. Span: A short multi-token span. These are usually used together with cloze prompts. Usually used for text classification, relation extraction and named entity recognition.
  3. Sentence: A sentence or document. These are commonly used with prefix prompts. Usually used for language generation and multiple-choice question answering.

Manual answer design

In manual answer design, both \(\mathcal{Z}\) and \(\mathcal{Y}\) are crafted manually. \(\mathcal{Z}\) can be the space of all tokens, fixed length spans, or token sequqnces. The most common mapping function to output \(\mathcal{Y}\) is the identity function. In many other cases, the space of possible outputs is constrained, especially for tasks with a limited label space such as text classification or entity recognition, or multiple- choice question answering.

Multi Prompt Learning

It has been demonstrated that the construction of multiple prompts can import the efficacy of prompting methods.

Prompt ensembling

Prompt ensembling combines multiple prompts into one single system. This strategy have three advantages: 1) leverage the complementary advantages of different prompts; 2) alleviate the cost of prompt engineering, since choosing one best-performing prompt is challenging; and 3) stabilize performance on downstream tasks. Methods to combine the predictions when using multiple prompts include uniform averaging, weighted averaging and majority voting.

Prompt augmentation

Prompt augmentation provides a few additional answered prompts (a.k.a. demonstrations) that can be used to demonstrate how the LM should fill the actual prompt instantiated with the input \(\textbf{x}\). For example, instead of just providing a prompt of “China’s capital is [Z] .”, the prompt can be prefaced by a few examples such as “Great Britain’s capital is London. Japan’s capital is Tokyo. China’s capital is [Z].” The efficacy of prompt augmentation can be explained by the observation that large pre-trained LMs are good at capturing repetitive patterns (Brown et al.).
